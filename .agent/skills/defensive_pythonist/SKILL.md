# Defensive Pythonist Skill

本技能专注于编写极致稳健的 Python 代码，特别是在处理不可靠的外部数据源（如股票行情、模型接口）时。

## 核心守则

### 1. 零信任输入 (Zero-Trust Input)
- **严格校验**：对所有外部 API 返回的数据（JSON/Dict）进行结构验证。严禁空指针引用（如直接访问 `json['data']['price']` 而不检查 `data` 是否存在）。
- **默认值策略**：在数据缺失时，必须提供合理的降级默认值，确保程序不崩溃。

### 2. 优雅降级 (Graceful Degradation)
- **异常捕获**：关键 I/O 必须包含针对网络波动、超时和 API 频率限制的捕获逻辑。
- **日志溯源**：所有捕获的错误必须记录足够的上下文信息，便于快速定位问题。

### 3. 类型暗示 (Type Safe)
- **类型提示**：强化静态类型检查，利用 Python 的 `typing` 模块减少运行期错误。

## 审计工作流
当被要求“使用防御性 Python 技能审计代码”时：
1. 识别代码中可能的“崩溃点”（如嵌套字典访问、未受保护的 I/O）。
2. 提供增加 `try-except-retry` 逻辑或 Pydantic 校验的重构方案。
